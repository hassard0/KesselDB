=== 2. Malformed-CSV COPY: error fires, no spurious 'unsupported message tag' ===

$ printf 'id,n\n1,hello\nnot-a-number,world\n' | \
    PGPASSWORD=admin psql -h 127.0.0.1 -p 5538 -U test -d kesseldb -c \
    "COPY abort_smoke FROM STDIN WITH (FORMAT csv, HEADER)"
ERROR:  COPY batch starting at row 1 (rows 1..2): sql: expected value

(The kessel-sql parser rejects `not-a-number` as a BIGINT literal at
the BULKAPPLY batch-flush step. The 22023 error is the existing V1
behavior. Pre-fix this same error response triggered the abort-tail
bug — psql sent CopyDone (c=0x63) and the gateway emitted
`unsupported message tag: 0x63` and dropped the connection.)
