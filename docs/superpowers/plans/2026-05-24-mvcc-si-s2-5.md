**GC + watermark recap (read once before T2).** Storage holds MVCC versions at 28-byte versioned keys `(type_id, object_id, inverted_commit_opnum)` per SP110. A logical key with N updates has N versions in storage. After any prefix of the log, the set of versions visible at `snapshot_opnum = S` is the subset with `commit_opnum <= S`. Once all live readers have `snapshot_opnum >= W` for some W, every version with `commit_opnum < W` is **obsolete** (no live reader can ever see it). S2.5 reclaims those obsolete versions via `Op::AdvanceWatermark { low_water_mark: W }` which the SM apply path runs deterministically. Once reclaimed, any future Tx with `snapshot_opnum < W` cannot be served correctly — S2.5 rejects them with `TxError::SnapshotTooOld { low_water_mark: W }` at Tx construction time.

**The SP113 supersession claim (the second headline).** SP113's `prune_pending_txs(pending_txs, current, MAX_TX_AGE = 4096)` evicts records older than `current - 4096`. Per the SP113 record's Decision 5 honest disclosure: a Tx whose snapshot is older than `current - MAX_TX_AGE` falls outside the lookback horizon; the SSI dangerous-structure detector cannot reach back to the relevant pending_txs records; an anomaly may be MISSED. S2.5's `prune_pending_txs_by_watermark(pending_txs, low_water_mark)` uses the WATERMARK instead of a fixed window — and the watermark, by Decision 2, is bounded above by min(active snapshots). Therefore any concurrent-Tx pair whose snapshots are within the active-reader population has its pending_txs records preserved across watermark advances; the dangerous-structure detector reaches them; the false-negative is closed. **T3's `it_supersedes_sp113_bounded_window_false_negative` constructs the SP113 PT-4 `too_old_snapshot_false_negative` scenario from `crates/kessel-storage/tests/pentest_mvcc_ssi.rs` and verifies the S2.5 path detects and aborts the previously-missed write-skew.**

**The SM apply seam (new `Op::AdvanceWatermark` arm).** The SM gains ONE NEW dispatch arm in `apply`:
1. Validate `low_water_mark > self.low_water_mark` (Decision 5) → on fail return `OpResult::WatermarkRejected { reason: NotMonotonic { proposed, current } }`.
2. Validate `low_water_mark <= self.commit_opnum` (Decision 5) → on fail return `OpResult::WatermarkRejected { reason: AboveCommitCeiling { proposed, current_commit } }`.
3. Call `mvcc::delete_versions_older_than(&mut self.store, low_water_mark)` — returns the count of versions deleted (Decision 3).
4. Call `ssi::prune_pending_txs_by_watermark(&mut self.pending_txs, low_water_mark)` — Decision 4 watermark-driven prune.
5. Update `self.low_water_mark = low_water_mark` (Decision 6).
6. Call `self.store.set_low_water_mark(low_water_mark)` so the Tx-side `Storage::low_water_mark()` accessor reflects the new value (Decision 7).
7. Return `OpResult::WatermarkAdvanced { new_low_water_mark: low_water_mark, versions_deleted, pending_txs_evicted }`.

The SP112 `Op::CommitTx` arm is UNCHANGED; SP113's SSI inner branch is UNCHANGED (it continues to call `ssi::prune_pending_txs(MAX_TX_AGE)` defensively on every commit).

```rust
pub fn delete_versions_older_than<V: Vfs>(
    store: &mut Storage<V>,
    low_water_mark: u64,
) -> Result<usize, MvccKeyError> {
    // Build the full versioned-key range. Lex order over 28-byte keys
    // is deterministic across replicas.
    let lo = vec![0x00u8; VERSIONED_KEY_LEN];
    let hi = vec![0xFFu8; VERSIONED_KEY_LEN];

    // Collect keys to delete in a Vec (avoid mutating during scan).
    let mut to_delete: Vec<Key> = Vec::new();
    for (k, _v) in store.scan_range_versions(&lo, &hi) {
        match decode_commit_opnum(&k) {
            Ok(c) if c < low_water_mark => to_delete.push(k),
            Ok(_) => continue,
            Err(_) => continue, // malformed key — skip
        }
    }

    let count = to_delete.len();
    // Delete in collected order (deterministic by scan-order).
    for k in to_delete {
        // SP110 storage uses op_number for any delete; pass the
        // low_water_mark itself (acts as the "apply timestamp" for the
        // GC; deterministic across replicas).
        store.delete(low_water_mark, k).map_err(|_| MvccKeyError::Length(0))?;
    }
    Ok(count)
}
```
