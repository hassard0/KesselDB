=== SP-Cloud-Deploy T3 — kind verify transcript (vulcan, 2026-05-30) ===

Verified end-to-end deploy of the SP-Cloud-Deploy Helm chart against
a real kind (Kubernetes-in-Docker) cluster on vulcan.

GHCR pull caveat: the published image at ghcr.io/hassard0/kesseldb:latest
is currently PRIVATE (default for new ghcr packages). The verify
sidestepped by building the image locally (`docker build -t
ghcr.io/hassard0/kesseldb:latest .` on vulcan) and side-loading it
into kind via `kind load docker-image`. Once the GHCR package is
flipped public (or pullSecrets are configured), the helm install
works as documented in NOTES.txt without intervention.

--- smoke: binary protocol via kessel CLI inside pod ---
$ kubectl exec deploy/kesseldb -- kessel --addr 127.0.0.1:6532 --token smoketest \
    'CREATE TABLE smoke (v U64 NOT NULL)'
OK  (table created, type_id=1)

$ kubectl exec deploy/kesseldb -- kessel --addr 127.0.0.1:6532 --token smoketest \
    'INSERT INTO smoke ID 1 (v) VALUES (42)'
OK

$ kubectl exec deploy/kesseldb -- kessel --addr 127.0.0.1:6532 --token smoketest \
    'SELECT SUM(v) FROM smoke'
= 42  (16 bytes)


--- smoke: HTTP gateway via port-forward ---
$ kubectl port-forward svc/kesseldb 16533:6533 &

$ curl -s -H 'Authorization: Bearer smoketest' http://127.0.0.1:16533/v1/health
{"status":"ok","primary":true,"view":0,"op_number":4,"role":"primary"}

$ curl -s -X POST -H 'Authorization: Bearer smoketest' \
       -H 'Content-Type: text/plain' \
       --data-binary 'SELECT * FROM smoke' \
       http://127.0.0.1:16533/v1/sql
{"status":"ok","bytes":36}
# 36 bytes = 4-byte LE length prefix + 32-byte encoded row (smoke{v=42}).
