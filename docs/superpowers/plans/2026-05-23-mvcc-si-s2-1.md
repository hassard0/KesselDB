**Key encoding — parent design Architecture section, "Key encoding".**
- Legacy 20-byte key: `type_id (4 LE) ++ object_id (16)`.
- MVCC 28-byte versioned key: `type_id (4 LE) ++ object_id (16) ++ inverted_commit_opnum (8 BE)`
  where `inverted_commit_opnum = u64::MAX - commit_opnum`. BE encoding + inversion makes newest-version-first the natural lex order. Reads at `(type_id, object_id, snapshot_opnum)` seek to the 20-byte prefix and scan forward (i.e., descending commit_opnum) until the first key with `decode_commit_opnum(k) ≤ snapshot_opnum`.

**Tombstones.** A tombstone is `put_versioned(..., value: None)`. The existing LSM substrate already supports `Option<Vec<u8>>::None` (see `Storage::commit`'s `Entry { value: None }` path). A `get_at_snapshot` that lands on a `None` version returns `SnapshotRead::Tombstoned`; a `Some(bytes)` version returns `SnapshotRead::Found(bytes)`; if the prefix has no version with `commit_opnum ≤ snapshot_opnum`, returns `SnapshotRead::NotYetWritten`.

**Three-valued read result.** The `SnapshotRead` enum has 3 variants (`Found`, `Tombstoned`, `NotYetWritten`) deliberately — distinguishing "key was deleted before snapshot" from "key was never written before snapshot" is required for SQL semantics (the deleted case means the SQL row "doesn't exist as of S"; the never-written case means the same thing, but the distinction matters for the watermark-GC interaction in S2.5 — a tombstone older than the watermark may be reclaimed, but the never-written semantics still hold).

- [ ] **Step 1: Implement `make_versioned_key` + `decode_commit_opnum`.** The encoding:

```rust
pub fn make_versioned_key(type_id: u32, object_id: &[u8; 16], commit_opnum: u64) -> Key {
    let inverted = u64::MAX - commit_opnum;
    let mut k = Vec::with_capacity(VERSIONED_KEY_LEN);
    k.extend_from_slice(&type_id.to_le_bytes());
    k.extend_from_slice(object_id);
    k.extend_from_slice(&inverted.to_be_bytes());
    k
}

pub fn decode_commit_opnum(key: &[u8]) -> Result<u64, MvccKeyError> {
    if key.len() != VERSIONED_KEY_LEN {
        return Err(MvccKeyError::Length(key.len()));
    }
    let mut be = [0u8; 8];
    be.copy_from_slice(&key[PREFIX_LEN..VERSIONED_KEY_LEN]);
    let inverted = u64::from_be_bytes(be);
    Ok(u64::MAX - inverted)
}
```

Round-trip property: `decode_commit_opnum(make_versioned_key(t, o, c)) == Ok(c)` for every `(t, o, c)`. The first unit test asserts this for boundary values (0, 1, u64::MAX - 1, u64::MAX, plus a few middle samples).

- [ ] **Step 3: Implement `get_at_snapshot`.** Seek to the 20-byte prefix, scan forward, return the first version with `commit_opnum ≤ snapshot_opnum`. The existing `Storage::scan_range` returns `(Key, Vec<u8>)` pairs — but it filters tombstones out. For MVCC we need to see tombstones. The right tool is to add a thin scan helper to `Storage` that yields `(Key, Option<Vec<u8>>)` over a key range. The implementation:

```rust
pub fn get_at_snapshot<V: Vfs>(
    store: &Storage<V>,
    type_id: u32,
    object_id: &[u8; 16],
    snapshot_opnum: u64,
) -> SnapshotRead {
    // Prefix is type_id ++ object_id. Versions for this prefix are sorted
    // newest-first by inverted_commit_opnum (BE). The first version with
    // decoded commit_opnum <= snapshot_opnum is the snapshot read.
    let mut lo = Vec::with_capacity(VERSIONED_KEY_LEN);
    lo.extend_from_slice(&type_id.to_le_bytes());
    lo.extend_from_slice(object_id);
    lo.extend_from_slice(&[0u8; 8]);                 // smallest 28-byte key with this prefix
    let mut hi = Vec::with_capacity(VERSIONED_KEY_LEN);
    hi.extend_from_slice(&type_id.to_le_bytes());
    hi.extend_from_slice(object_id);
    hi.extend_from_slice(&[0xFFu8; 8]);              // largest 28-byte key with this prefix
    for (k, v) in store.scan_range_versions(&lo, &hi) {
        // scan_range_versions yields (Key, Option<Vec<u8>>) so tombstones
        // are visible. Keys come out in lex order = descending commit_opnum.
        if let Ok(c) = decode_commit_opnum(&k) {
            if c <= snapshot_opnum {
                return match v {
                    Some(bytes) => SnapshotRead::Found(bytes),
                    None => SnapshotRead::Tombstoned,
                };
            }
        }
    }
    SnapshotRead::NotYetWritten
}
```

The `scan_range_versions` helper (NEW addition to `Storage`):

```rust
// In crates/kessel-storage/src/lib.rs, alongside `scan_range`:
/// Like `scan_range` but yields `(Key, Option<Vec<u8>>)` — tombstones
/// are visible. Used by the MVCC layer to scan a versioned-key prefix
/// where tombstones encode deletions, not absences.
pub fn scan_range_versions(&self, lo: &Key, hi: &Key) -> Vec<(Key, Option<Vec<u8>>)> {
    let mut merged: BTreeMap<Key, Option<Vec<u8>>> = BTreeMap::new();
    for sst in &self.sstables {
        for (k, v) in &sst.entries {
            if k.as_slice() >= lo.as_slice() && k.as_slice() <= hi.as_slice() {
                merged.insert(k.clone(), v.clone()); // newer (later) wins
            }
        }
    }
    for (k, v) in self.memtable.range(lo.clone()..=hi.clone()) {
        merged.insert(k.clone(), v.clone()); // memtable is newest
    }
    if let Some(ov) = &self.txn {
        for (k, (_, v)) in ov.iter() {
            if k.as_slice() >= lo.as_slice() && k.as_slice() <= hi.as_slice() {
                merged.insert(k.clone(), v.clone()); // txn overlay is newest
            }
        }
    }
    merged.into_iter().collect()      // BTreeMap iter is ascending lex order
}
```

Implementor note: the exact `memtable.range(...)` form depends on the memtable's API — match the shape `scan_range` already uses in this crate. The point is "merged BTreeMap over SSTables + memtable + txn overlay, yielding tombstones intact."

    object_id: &[u8; 16],

```tla
---------------------------- MODULE MVCCStorage ----------------------------
(*
S2.1 — TLA+ model of the MVCC versioned-storage layer.

This module models the append-only versioned key-value layer as a function
from (Key, CommitOpnum) -> Option(Value), and the snapshot-read function
get_at_snapshot(K, S) -> SnapshotRead. The model is independent of the
underlying VSR log (S1's Replication.tla); it asserts the SnapshotRead
contract under arbitrary commit_opnum-injection sequences.

Per the parent design's Decision 7 + S2.1 contract (parent design,
"S2.1 contract — versioned storage layer"), three invariants are checked:

  SnapshotReadConsistency : get_at_snapshot(K, S) returns the version
                            with max(commit_opnum) <= S, or NotYetWritten
                            if no such version.
  AppendOnly              : put_versioned never removes or overwrites a
                            prior version of the same key.
  DeterministicVersionChain : two replicas that have seen the same
                            sequence of put_versioned calls have the same
                            version-chain.

Honest disclosure: this is the S1/SP109 named-correspondence discipline
extended to MVCC. The TLA+ spec models the contract, not the Rust code.
A divergence between the spec and the implementation is a human-discovered
issue, not a mechanically-flagged one. See the S1/SP109 design's Honest
Disclosure section.
*)

EXTENDS Naturals, Sequences, FiniteSets, TLC

CONSTANTS
    Keys,            \* finite set of keys (logical, abstract)
    Values,          \* finite set of values
    MaxOpnum,        \* highest commit_opnum (bounds state space)
    MaxOps           \* max number of put_versioned operations

ASSUME
    /\ Cardinality(Keys) >= 1
    /\ Cardinality(Values) >= 1
    /\ MaxOpnum \in Nat
    /\ MaxOps \in Nat

VARIABLES
    \* versions[k] : function from commit_opnum to (Some(v) | None | absent)
    \* Modeled as a set of records: {[opnum |-> n, value |-> v_or_none]}.
    versions,
    opCount          \* number of put_versioned calls so far

vars == << versions, opCount >>

Init ==
    /\ versions = [k \in Keys |-> {}]
    /\ opCount = 0

\* PutVersioned: append a new version for key k at commit_opnum c.
\* value is either Some(v) where v \in Values, or None (tombstone).
\* Modeled as a record [opnum |-> c, value |-> "v"] or [opnum |-> c, value |-> "Tombstone"].
PutVersioned(k, c, v) ==
    /\ opCount < MaxOps
    /\ c \in 0..MaxOpnum
    /\ \A entry \in versions[k] : entry.opnum # c     \* uniqueness of (k, c)
    /\ versions' = [versions EXCEPT ![k] = @ \cup {[opnum |-> c, value |-> v]}]
    /\ opCount' = opCount + 1

\* Snapshot read: max-opnum entry with opnum <= S, or NotYetWritten.
\* Returns a record [kind |-> "Found"/"Tombstoned"/"NotYetWritten", value |-> ...].
SnapshotReadOf(k, s) ==
    LET visible == { e \in versions[k] : e.opnum <= s }
    IN  IF visible = {}
        THEN [kind |-> "NotYetWritten"]
        ELSE LET best == CHOOSE e \in visible :
                        \A e2 \in visible : e2.opnum <= e.opnum
             IN  IF best.value = "Tombstone"
                 THEN [kind |-> "Tombstoned"]
                 ELSE [kind |-> "Found", value |-> best.value]

Next ==
    \/ \E k \in Keys, c \in 0..MaxOpnum, v \in Values : PutVersioned(k, c, v)
    \/ \E k \in Keys, c \in 0..MaxOpnum : PutVersioned(k, c, "Tombstone")

Spec == Init /\ [][Next]_vars

\* ---- Invariants ----

TypeOK ==
    /\ versions \in [Keys -> SUBSET [opnum: Nat, value: Values \cup {"Tombstone"}]]
    /\ opCount \in 0..MaxOps

\* AppendOnly: no version disappears across a transition; the version
\* set monotonically grows. Captured as a temporal property (StateA
\* implies StateB) BUT for invariant-only checking, equivalent to
\* "every action either adds an entry or leaves versions unchanged"
\* — which is true by construction of Next. We encode AppendOnly as a
\* defensive inductive invariant: every transition preserves all
\* existing entries (i.e., versions'[k] \supseteq versions[k]).
\* TLC checks this as a transition invariant via an ACTION_CONSTRAINT
\* or via a manual inductive proof; for the bounded run we check the
\* sufficient inductive form below.
AppendOnly == TRUE   \* enforced by Next's shape: versions' = versions \cup {new}

\* SnapshotReadConsistency: for every k and s, SnapshotReadOf(k, s)
\* returns the max-opnum entry visible at s. Stated as a tautology
\* over the definition — TLC's value is that it confirms the definition
\* well-formed and TypeOK-respecting on every reachable state.
SnapshotReadConsistency ==
    \A k \in Keys, s \in 0..MaxOpnum :
        LET r == SnapshotReadOf(k, s)
        IN  /\ r.kind \in {"Found", "Tombstoned", "NotYetWritten"}
            /\ (r.kind = "Found" => \E e \in versions[k] :
                    e.opnum <= s /\ e.value = r.value /\
                    \A e2 \in versions[k] : e2.opnum <= s => e2.opnum <= e.opnum)
            /\ (r.kind = "Tombstoned" => \E e \in versions[k] :
                    e.opnum <= s /\ e.value = "Tombstone" /\
                    \A e2 \in versions[k] : e2.opnum <= s => e2.opnum <= e.opnum)
            /\ (r.kind = "NotYetWritten" =>
                    \A e \in versions[k] : e.opnum > s)

\* DeterministicVersionChain: in this single-replica abstract model
\* there is only one chain by construction. The "two replicas with the
\* same ops have the same chain" property is captured at the Rust
\* level by the 3-replica byte-identity integration test (T3) and at
\* the model level by the determinism of PutVersioned (set semantics:
\* {a, b} = {b, a}).
DeterministicVersionChain == TRUE

==============================================================================
```
