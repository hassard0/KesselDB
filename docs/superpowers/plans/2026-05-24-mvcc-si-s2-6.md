**SQL→MVCC mapping table (read once before T2).** The 14 data-row SM apply arms map to MVCC primitives as follows:

| Legacy Op apply arm | MVCC primitive(s) used | Notes |
|---|---|---|
| `Op::Create { type_id, id, record }` | `mvcc::put_versioned(&mut store, op_number, type_id, id, Some(record))` | tombstone path: `Op::Delete` puts `None` value |
| `Op::Update { type_id, id, record }` | `mvcc::put_versioned(&mut store, op_number, type_id, id, Some(record))` | overwrite via new version |
| `Op::Delete { type_id, id }` | `mvcc::put_versioned(&mut store, op_number, type_id, id, None)` | tombstone version |
| `Op::GetById { type_id, id }` | `mvcc::get_at_snapshot(&store, type_id, id, snapshot_opnum)` | snapshot = current_commit_opnum |
| `Op::Select { type_id, program, limit }` | `mvcc::scan_at_snapshot(&store, type_id, snapshot_opnum)` + filter via VM | NEW primitive; full-type scan returning latest <= snapshot per object_id |
| `Op::QueryRows { type_id, eq_preds, program, limit, range_preds }` | index-narrowed candidates via index keyspace (UNCHANGED — Decision 1 scope) → `mvcc::get_at_snapshot` for each | indexes stay 20-byte; data row reads through MVCC |

- [ ] **Step 2: Add the 4 accessor methods.** After `Storage::set_low_water_mark` accessor wiring (SP114 pattern), add to `impl<V: Vfs> StateMachine<V>`:

```rust
/// SP115 / S2.6 (Decision 7): Register a new auto-commit Tx's
/// snapshot_opnum in the active-snapshots multiset. Called by the
/// server's apply_one immediately before Tx::begin/begin_rw/begin_ssi.
///
/// Idempotent w.r.t. multiset semantics — same snapshot called twice
/// increments the count to 2; two `unregister_snapshot(s)` calls
/// remove it.
pub fn register_snapshot(&mut self, snapshot_opnum: u64) {
    *self.active_snapshots.entry(snapshot_opnum).or_insert(0) += 1;
}

/// SP115 / S2.6 (Decision 7): Unregister an auto-commit Tx's
/// snapshot_opnum. Called by the server's apply_one at Tx::commit /
/// abort / commit_read_only. Decrements the count; removes the key
/// at count = 0.
///
/// If the snapshot was never registered (count = 0), this is a
/// no-op — defensive against caller mismatch but should never happen
/// in correctly-wired code (T5 pentest exercises the edge).
pub fn unregister_snapshot(&mut self, snapshot_opnum: u64) {
    use std::collections::btree_map::Entry;
    match self.active_snapshots.entry(snapshot_opnum) {
        Entry::Occupied(mut e) => {
            let v = e.get_mut();
            *v = v.saturating_sub(1);
            if *v == 0 {
                e.remove_entry();
            }
        }
        Entry::Vacant(_) => {} // defensive no-op
    }
}

/// SP115 / S2.6 (Decision 6): The minimum active snapshot_opnum
/// across all currently-registered auto-commit Tx. Returns `None`
/// when no Tx is active (heartbeat producer interprets None as
/// "watermark may advance to current_commit_opnum"). Deterministic
/// at the BTreeMap iteration level.
pub fn min_active_snapshot(&self) -> Option<u64> {
    self.active_snapshots.keys().next().copied()
}

/// SP115 / S2.6 (Decision 3): The latest committed op_number visible
/// to the SM. Auto-commit Tx reads this at Tx::begin to pin its
/// snapshot at READ COMMITTED. Deterministic: equal across replicas
/// at the same log prefix.
///
/// Implementation note: returns the SM's authoritative "highest applied
/// op_number" tracker. Verify the field at impl-time (likely
```

```rust
pub fn scan_at_snapshot<V: Vfs>(
    store: &Storage<V>,
    type_id: u32,
    snapshot_opnum: u64,
) -> Vec<(crate::ObjectId, Vec<u8>)> {
    // Build the type-prefixed range: lo = (type_id, 0u128, 0u64);
    // hi = (type_id+1, 0u128, 0u64) excluded → or full-end if
    // type_id == u32::MAX. The exact byte construction matches
    // SP110's make_versioned_key for the lo bound; the hi bound is
    // the next-type prefix.
    let mut lo = Vec::with_capacity(VERSIONED_KEY_LEN);
    lo.extend_from_slice(&type_id.to_be_bytes());
    lo.extend_from_slice(&[0u8; 16]); // object_id all-zeros
    lo.extend_from_slice(&[0u8; 8]); // inverted_commit_opnum all-zeros
    
    let mut hi = Vec::with_capacity(VERSIONED_KEY_LEN);
    hi.extend_from_slice(&type_id.saturating_add(1).to_be_bytes());
    hi.extend_from_slice(&[0u8; 16]);
    hi.extend_from_slice(&[0u8; 8]);
    
    // Per-object_id "latest <= snapshot" reduction. Iterate the
    // sorted range; group by object_id; for each group, keep the
    // FIRST version whose commit_opnum <= snapshot (inverted_commit_opnum
    // makes "first" = "largest commit_opnum" = latest).
    let mut out: Vec<(crate::ObjectId, Vec<u8>)> = Vec::new();
    let mut current_oid: Option<[u8; 16]> = None;
    let mut current_taken: bool = false;
    
    for (k, v) in store.scan_range_versions(&lo, &hi) {
        if k.len() != VERSIONED_KEY_LEN { continue; }
        let oid: [u8; 16] = k[4..20].try_into().unwrap();
        let commit_opnum = match decode_commit_opnum(&k) {
            Ok(c) => c,
            Err(_) => continue,
        };
        
        if Some(oid) != current_oid {
            // New object_id group
            current_oid = Some(oid);
            current_taken = false;
        }
        
        if current_taken { continue; }
        
        if commit_opnum <= snapshot_opnum {
            // First (latest) visible version for this object_id
            // (since the scan is sorted by inverted_commit_opnum
            // within object_id, the first version <= snapshot_opnum
            // we encounter IS the latest visible).
            current_taken = true;
            // Skip tombstoned entries (value empty or None marker —
            // verify SP110 convention)
            if !v.is_empty() {
                out.push((crate::ObjectId::from_bytes(oid), v));
            }
        }
    }
    
    out
}
```
