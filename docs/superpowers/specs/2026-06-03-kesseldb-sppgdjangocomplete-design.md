# SP-PG-DJANGO-COMPLETE — Django ORM full CRUD (IDENTITY + aggregate render) — design

Date: 2026-06-03

## 1. Context

The `SP-PG-SQL-QUOTED-IDENT` arc took the Django 6 ORM smoke from 2/8 to
6/8 by teaching the kessel-sql lexer SQL-standard double-quoted
identifiers. Two precisely-diagnosed gaps remain (per
`docs/superpowers/sppgormdjango-smoke-2026-06-03.txt`), and closing BOTH
takes Django to full CRUD 8/8 — a second production ORM (after
SQLAlchemy 7/7) fully working against KesselDB.

The two gaps:

1. **SP-PG-DDL-IDENTITY** (blocks stage `schema_create`): Django 6's
   default `BigAutoField` renders the PK DDL as
   `"id" bigint NOT NULL PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY`
   — the SQL-standard autoincrement spelling, NOT `BIGSERIAL`.
   kessel-sql's CREATE TABLE column parser does not recognize the
   `GENERATED … AS IDENTITY` column-constraint run.

2. **SP-PG-SQL-AGG-ALIAS-RENDER** (blocks the final `.count()` in stage
   `orm_delete`): Django emits `SELECT COUNT(*) AS "__count" FROM "t"`
   — an aliased aggregate over a FROM table. kessel-sql parses
   `COUNT(*)` but DROPS the alias; the gateway's render path has no
   case for a FROM-ful scalar aggregate (`select_star_table` and
   `select_columns` both return None for it, so `render_select_got`
   falls into the "0A000 V1 PG-wire only renders SELECT *" error arm).

## 2. Feature 1 — `GENERATED … AS IDENTITY` (reuses SP-PG-SERIAL machinery)

In the CREATE TABLE column-definition loop (`compile` in
`crates/kessel-sql/src/lib.rs`), the per-column modifier run currently
handles `NOT NULL`, an inline `PRIMARY KEY`, and `DEFAULT <lit>` in a
fixed sequence. Django's column emits the run
`NOT NULL PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY`, so the modifier
parsing is restructured into an **order-independent loop** that accepts:

- `NOT NULL` → sets `nullable = false`
- `PRIMARY KEY` → records the PK column name, sets `nullable = false`
- `GENERATED { ALWAYS | BY DEFAULT } AS IDENTITY [ ( seq_options ) ]`
  → marks the column SERIAL (adds to `serial_col_names`, same flag
  BIGSERIAL/SERIAL would set). The optional `( … )` sequence-options
  group (`START WITH n`, `INCREMENT BY n`, …) is parsed-and-IGNORED in
  V1 — KesselDB's deterministic counter starts at 1, increments by 1
  (named follow-up `SP-PG-IDENTITY-SEQOPTS` if a model ever relies on a
  custom start/increment; Django's default `BigAutoField` does not).
- `DEFAULT <lit>` → unchanged.

A column declared `bigint … GENERATED … AS IDENTITY` keeps its declared
type (`bigint` → I64) and is added to `serial_col_names`. The existing
`serial_field_id` computation (a column that is BOTH a PK and serial)
then flags it as the deterministic autoincrement PK — **identical** to
the path `id BIGSERIAL PRIMARY KEY` already takes. No SM, catalog, or
proto change: IDENTITY is a pure parser-front alias onto the proven
`SP-PG-SERIAL-RETURNING` counter (digest-covered, apply-thread,
3-replica byte-identical).

Determinism: unchanged. The autoincrement value is assigned by the SM's
existing `SERIAL_SENTINEL` path on the apply thread; IDENTITY only
changes which SQL spelling sets the `serial_pk` catalog flag at DDL
parse time. The counter, sentinel, and digest coverage are byte-for-byte
the SP-PG-SERIAL machinery.

## 3. Feature 2 — aggregate alias + FROM-ful render

### 3a. kessel-sql parse — capture the alias

`parse_agg` (inside `compile_select`) currently parses
`AGG ( * | col )` and discards anything after the `)`. Add an
optional `AS <ident>` capture so `COUNT(*) AS "__count"` parses cleanly
(the quoted alias already lexes as an `Ident` after the quoted-ident
arc). The alias is stored on `AggSpec` but does NOT change the emitted
`Op` (the Op proto carries no output name); it exists so a sibling
`select_aggregate` text-helper can name the RowDescription column. The
single-aggregate, no-leading-col, no-GROUP-BY case still emits a
byte-identical `Op::Aggregate`.

### 3b. kessel-sql — `select_aggregate` text helper

New `pub fn select_aggregate(sql) -> Option<SelectAgg>` (mirrors
`select_columns` / `select_star_table`): lexer-backed detector that
matches `SELECT <AGG>( * | [t.]col ) [AS alias] FROM <table>` (single
aggregate, no GROUP BY, no JOIN, no extra projection) and returns
`SelectAgg { table, kind, field: Option<String>, alias: Option<String> }`.
`None` for anything else (multi-aggregate, GROUP BY, etc. — those keep
their existing paths). The gateway uses this to render the scalar.

### 3c. gateway — render the scalar aggregate

The engine's `Op::Aggregate` returns `OpResult::Got(bytes)` where
`bytes` is a 16-byte little-endian `i128` (the scalar fold — confirmed
in `kessel-sm` + `kesseldb-server/sharded_engine.rs`). In the SELECT
`OpResult::Got` arm of `dispatch_simple_query` (and the equivalent
extended-query path if present), BEFORE the existing `render_select_got`
shapes, check `kessel_sql::select_aggregate(sql)`. If Some:

- column name = `alias` if present, else the lowercase function name
  (`count` / `sum` / `min` / `max` / `avg`) — matching PostgreSQL's
  default output column naming.
- type_oid = `int8` (20) — the i128 scalar is rendered as a signed
  decimal; COUNT/SUM/MIN/MAX of integer columns all fit the bigint text
  rendering KesselDB already uses for I64.
- decode the 16-byte LE i128, render its decimal ASCII as ONE DataRow,
  ONE column.
- emit RowDescription(1 col) + DataRow(value) + CommandComplete("SELECT
  1") + ReadyForQuery.

If `Got` is not 16 bytes (defensive), fall through to the existing
render path.

Read-only: the aggregate is a pure read; no determinism surface is
touched. The render is a deterministic decode of the engine's bytes.

## 4. Scope (V1) / out-of-scope follow-ups

In scope: `GENERATED {ALWAYS|BY DEFAULT} AS IDENTITY [(...)]` as a
serial-PK alias; single scalar aggregate `AGG(...) [AS alias] FROM t`
parse + render with the alias (or default) column name.

Named follow-ups (NOT this arc):
- `SP-PG-IDENTITY-SEQOPTS` — honor `START WITH` / `INCREMENT BY`.
- `SP-PG-AGG-MULTI-RENDER` — render a multi-column / GROUP BY aggregate
  result over the wire (the Op exists; only the single-scalar wire
  render lands here).
- `SP-PG-AGG-FLOAT` — AVG as a real fractional result (V1 i128 is
  integer-truncated, matching the existing engine fold).
