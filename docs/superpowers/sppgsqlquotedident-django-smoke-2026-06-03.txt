RESIDUAL GAPS (both are pre-named follow-ups, NOT quoting)
==========================================================
1. schema_create — `expected ',' or ')'`. The quoted identifiers parse
   fine; the trip is Django 6's default PK DDL spelling
   `"id" bigint NOT NULL PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY`.
   Follow-up: SP-PG-DDL-IDENTITY (accept GENERATED [BY DEFAULT|ALWAYS]
   AS IDENTITY as a BIGSERIAL-autoincrement alias).

2. orm_delete — `expected FROM`. The DELETE itself PASSES; the stage
   runs a trailing `Author.objects.count()` →
   `SELECT COUNT(*) AS "__count" FROM "t"`. Direct probe (psycopg, same
   server) disambiguates:
       OK    DELETE FROM "dt" WHERE "dt"."id" IN (%s)         -> (no rows)
       FAIL  SELECT COUNT(*) AS "__count" FROM "dt"           -> expected `FROM`
       FAIL  SELECT COUNT(*) FROM "dt"                        -> V1 PG-wire
             only renders SELECT * or an explicit projection list
   Follow-up: SP-PG-SQL-AGG-ALIAS-RENDER (parse `COUNT(*) AS <alias>`
   incl. the quoted alias + PG-wire render of a FROM-ful aggregate
   projection). Django `.count()`/`.exists()`/`.aggregate()` all need it.
